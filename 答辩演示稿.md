# 街舞信息共享平台答辩演示稿

## 1. 项目开场介绍

本项目名称为“基于 JavaScript 的街舞信息共享平台设计与实现”。  
项目面向街舞爱好者、舞者、活动主办方和相关商家，目标是解决街舞领域活动信息、作品展示、社群交流和周边服务分散的问题，构建一个集活动、视频、社交、商城和个人中心于一体的垂直平台。

系统技术方案如下：

- 前端：JavaScript、HTML5、CSS3、React、Vite
- 后端：Python、Django
- 数据库：SQLite

## 2. 系统演示流程

### 2.1 启动系统

先展示项目可以正常启动：

- 启动 Django 后端服务
- 启动 React 前端服务
- 打开浏览器访问前端首页

说明：

- 后端负责业务接口和数据管理
- 前端负责页面展示和交互
- 当前系统已支持完整的前后端联调

### 2.2 首页演示

进入首页后，介绍平台总体结构：

- 首页展示平台简介
- 展示聊天室分类
- 展示最新视频
- 展示商城商品
- 展示平台当前开发状态

答辩时可强调：首页主要用于集中展示平台核心内容入口，帮助用户快速进入活动、视频、社交和商城等业务场景。

### 2.3 用户登录与注册演示

进入登录页面，演示：

- 用户注册
- 用户登录
- 登录后页面状态变化
- 退出登录

建议使用演示账号：

- `bboy_chen / Dance123456`
- `jazz_luna / Dance123456`

答辩时可说明：系统基于 Django Session 实现登录认证，未登录用户无法进行发布、报名、评论、购物等操作。

### 2.4 活动模块演示

进入活动页面，演示：

- 查看活动列表

## 3. 关键设计说明

### 3.1 系统架构

系统采用前后端分离架构：

- React 负责视图层与页面交互
- Django 负责接口逻辑、数据模型和后台管理
- SQLite 负责结构化数据存储

该架构适合毕业设计阶段的快速实现与演示。

### 3.2 数据设计

系统围绕以下核心实体展开：

- 用户
- 活动
- 活动报名
- 视频
- 视频评论
- 聊天室
- 聊天消息
- 商品
- 购物车
- 订单
- 收藏
- 关注

### 3.3 项目亮点

- 业务覆盖较完整，包含街舞活动、作品、社交和商城多个场景
- 前后端联调完成，能够真实演示平台业务流程
- 模块边界清晰，便于后续继续扩展
- 提供演示数据命令，适合答辩现场快速复现

## 4. 可继续优化的方向

答辩时如果老师提问“后续还能怎么优化”，可从以下方向回答：

- 接入真实图片和视频上传
- 聊天模块升级为 WebSocket 实时通信
- 商城模块增加订单取消、发货状态等流程
- 活动模块增加地图定位和附近活动推荐
- 视频模块增加标签分类、推荐排序和搜索优化
- 系统部署到云服务器，实现公网访问
